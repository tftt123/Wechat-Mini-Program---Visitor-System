# 访客管理系统（微信小程序）

## 项目简介

基于微信小程序 + 微信云开发的访客管理系统，覆盖访客从预约到离厂的全流程管理。

## 核心流程

1. **员工**在小程序生成6位邀请码，发送给访客
2. **访客**到达门卫处，扫描固定QR Code进入小程序
3. **访客**输入邀请码，填写个人信息并提交
4. **被访员工**在小程序确认拜访
5. **门卫**在系统中确认访客入厂
6. **访客**离开前，被访员工带访客到门卫处，在小程序做物品确认
7. **门卫**确认物品后，点"确认出"，访客离厂

## 角色权限

| 角色 | 权限 |
|------|------|
| 访客 | 扫码进入，输入邀请码，填写个人信息 |
| 员工 | 生成邀请码，确认拜访，物品确认 |
| 门卫 | 查看当日来访，确认入厂/出厂 |
| 人事 | 查看全部拜访记录，统计查询 |

## 技术栈

- **前端**：微信小程序原生（WXML + WXSS + JS）
- **后端**：微信云开发（云函数 + 云数据库）
- **通知**：微信小程序订阅消息

## 部署步骤

### 1. 注册微信小程序

- 访问 [微信公众平台](https://mp.weixin.qq.com/)
- 注册小程序账号（选择企业主体）
- 获取 AppID

### 2. 开通微信云开发

- 登录小程序后台 → 云开发 → 开通
- 记录云开发环境ID

### 3. 导入项目

- 下载并安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
- 打开微信开发者工具 → 导入项目
- 选择 `visitor-system` 文件夹
- 填入 AppID
- 勾选"使用云服务"

### 4. 修改配置

编辑 `miniprogram/app.js`，将 `your-cloud-env-id` 替换为您的云开发环境ID：

```javascript
wx.cloud.init({
  env: 'your-cloud-env-id', // ← 替换为您的环境ID
  traceUser: true
})
```

编辑 `project.config.json`，将 `wxYOURAPPID` 替换为您的小程序 AppID。

### 5. 创建数据库集合

在云开发控制台 → 数据库中创建以下集合：

- `users`
- `visits`
- `visit_logs`
- `notifications`

为以下字段创建索引：
- `users`: `phone`（唯一）
- `visits`: `inviteCode`（唯一）
- `visits`: `employeeId`
- `visits`: `status`
- `visits`: `visitDate`
- `notifications`: `recipientId`

### 6. 部署云函数

在微信开发者工具中：

1. 右键 `cloudfunctions/auth` → 创建并部署：云端安装依赖
2. 右键 `cloudfunctions/invites` → 创建并部署：云端安装依赖
3. 右键 `cloudfunctions/visits` → 创建并部署：云端安装依赖
4. 右键 `cloudfunctions/notifications` → 创建并部署：云端安装依赖

### 7. 生成固定QR Code

- 在小程序后台 → 工具 → 生成小程序码
- 页面路径填写：`pages/index/index`
- 下载后打印，贴在门卫处

### 8. 配置订阅消息（可选）

- 小程序后台 → 功能 → 订阅消息
- 搜索并添加模板：访客通知类
- 在代码中替换模板ID（如需使用）

## 使用说明

### 首次使用

1. 员工/人事/门卫在"我的"页面注册账号（选择对应角色）
2. 注册后即可使用对应功能

### 员工发起拜访

1. 进入"员工端"页面
2. 点击"生成邀请码"
3. 填写来访日期、时间和目的
4. 将生成的6位邀请码告知访客

### 访客登记

1. 到达门卫处，扫描墙上QR Code
2. 输入邀请码
3. 填写个人信息（姓名、手机号、公司等）
4. 提交后等待被访人确认

### 门卫操作

1. 进入"门卫端"页面
2. "待入厂"列表中查看已确认的访客
3. 核实身份后点击"确认入厂"
4. "待出厂"列表中查看已做物品确认的访客
5. 核实后点击"确认出厂"

## 项目结构

```

├── cloudfunctions/          # 云函数
│   ├── auth/                # 认证（注册/登录）
│   ├── invites/             # 邀请码管理
│   ├── visits/              # 拜访记录管理
│   └── notifications/       # 通知推送
├── miniprogram/             # 小程序前端
│   ├── pages/               # 页面
│   │   ├── index/           # 访客扫码入口
│   │   ├── visitor-form/    # 访客填表
│   │   ├── employee/        # 员工端
│   │   ├── security/        # 门卫端
│   │   ├── hr/              # 人事端
│   │   └── mine/            # 我的（登录/注册）
│   ├── services/            # API封装
│   ├── utils/               # 工具函数
│   └── app.js               # 小程序入口
└── project.config.json      # 项目配置
```

## 注意事项

1. 本项目使用手机号直接登录（演示用途），生产环境建议接入短信验证码
2. 订阅消息需要用户主动授权，首次使用时会引导订阅
3. 邀请码有效期为7天，过期后需重新生成
4. 数据权限：员工只能查看自己的拜访记录，门卫只能查看当日记录
