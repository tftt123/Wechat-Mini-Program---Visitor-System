## 一、整体设计规范

### 1.1 色彩体系

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-primary` | `#1A2A3A` | 主色调：导航栏、主按钮、页面标题、核心文字 |
| `--color-primary-hover` | `#0F1A25` | 主色 hover/pressed 状态 |
| `--color-secondary` | `#2C3E50` | 次级色：副标题、次级按钮、标签文字 |
| `--color-surface` | `#FFFFFF` | 卡片背景、表单背景、浮层背景 |
| `--color-background` | `#F5F7FA` | 页面全局背景 |
| `--color-muted` | `#E8ECF0` | 分割线、禁用态背景、轻灰填充 |
| `--color-border` | `#D1D9E0` | 卡片边框、输入框边框、列表分割线 |
| `--color-text-primary` | `#1A2A3A` | 主文字（标题、正文） |
| `--color-text-secondary` | `#4A5568` | 次级文字（描述、辅助信息） |
| `--color-text-muted` | `#94A3B8` | 占位符、禁用文字、时间戳 |
| `--color-brand` | `#D32F2F` | **品牌红：仅用于强调，严禁大面积铺陈** |
| `--color-brand-hover` | `#B71C1C` | 品牌红 hover 态 |
| `--color-brand-light` | `#FFEBEE` | 品牌红 5% 浅底，用于角标/提示背景 |
| `--color-success` | `#16A34A` | 成功状态、正向操作 |
| `--color-success-light` | `#DCFCE7` | 成功态浅底 |
| `--color-warning` | `#CA8A04` | 警告状态 |
| `--color-warning-light` | `#FEF9C3` | 警告态浅底 |
| `--color-danger` | `#DC2626` | 危险操作：删除、作废、拒绝 |
| `--color-danger-light` | `#FEE2E2` | 危险态浅底 |

**色彩使用原则：**
- 页面 80% 以上为中性色（白/灰/深灰）
- 红色出现面积不超过单屏 3%
- 信息层级通过明度对比建立，而非色相跳跃

## 二、红色（品牌色）使用规则

### 绝对可用场景

| 场景 | 具体组件 | 使用方式 |
|------|----------|----------|
| 危险操作 | 删除按钮、作废按钮、拒绝按钮 | `btn-danger`：红底白字 |
| 未读角标 | Tab 徽章、通知红点 | `#D32F2F` 实心圆，直径 32rpx |
| 紧急状态 | "待出厂"标签 | 红字 + 浅红底 pill |
| 关键节点 Icon | 回收工牌、警告 Icon | `#D32F2F` 描边或填充 Icon |
| 表单错误 | 字段校验失败提示 | `#DC2626` 文字 + 左侧红色竖线 |
| 品牌点缀 | Logo、顶部导航标题前的品牌竖线 | 6rpx 宽品牌红色块 |

### 绝对禁止场景

| 场景 | 原因 |
|------|------|
| 页面全局背景 | 破坏企业级沉稳感，造成视觉焦虑 |
| 导航栏背景 | 喧宾夺主，干扰信息阅读 |
| 普通列表项背景 | 大面积红色导致信息层级混乱 |
| 主按钮默认态（非危险操作） | 普通提交/确认应使用深色主按钮 |
| 卡片整体边框 | 红色边框会吸引过多注意力 |
| Tab 栏激活态背景 | Tab 激活应使用深色文字 + 底部指示线 |

### 红色面积控制原则

```
单屏面积占比 ≤ 3%
├── 状态标签：每个标签面积占比 ≈ 0.2%
├── 操作按钮：单个按钮面积占比 ≈ 1.5%
├── 未读角标：每个角标面积占比 ≈ 0.05%
└── 品牌竖线：每条 6rpx × 36rpx ≈ 0.01%
```

## 四、修改前 vs 修改后 对比说明

### 全局层面

| 维度 | 修改前 | 修改后 | 企业级提升 |
|------|--------|--------|-----------|
| **主色调** | `#C62828` 品牌红大面积使用 | `#1A2A3A` 深灰蓝为主 | 从"醒目"转向"沉稳"，传递专业管理气质 |
| **状态标签** | 全部红底白字 | 语义化配色（灰/黄/绿/红） | 信息层级清晰，红色仅用于真正紧急的状态 |
| **背景** | `#F5F5F5` 偏暖灰 | `#F5F7FA` 偏冷灰 | 冷灰更显科技感与秩序感 |
| **卡片** | 大圆角 24rpx + 无阴影 | 适中圆角 12rpx + 轻阴影 | 减少装饰感，增加层级区分 |
| **按钮** | 红色主按钮泛滥 | 深色主按钮 + 红色仅危险操作 | 操作语义明确，降低误操作焦虑 |
| **字体层级** | 层次弱，大小差异小 | 6 级明确层级（H1→Overline） | 信息扫描效率提升 |

### 页面层面

| 页面 | 修改前问题 | 修改后亮点 |
|------|-----------|-----------|
| **邀请码生成** | 邀请码字符小、无视觉锚点 | 大字符+浅灰底卡片+字间距，邀请码成为视觉焦点 |
| **访客填表** | 表单无分组，错误提示突兀 | 信息卡片（只读）+ 输入卡片分离，错误提示带左红线 |
| **被访人确认** | 信息平铺，按钮同色 | Timeline 处理记录 + 语义化按钮层级 |
| **门卫列表** | Tab 反馈弱，列表松散 | 深灰激活 Tab + 高密度信息卡片 + 角标引导 |
| **物品确认** | 上传区边界模糊 | 虚线边框上传区 + 缩略图网格 + 删除角标 |
| **回收工牌** | 无独立视觉节点 | 独立卡片+品牌红 Icon+绿色确认按钮，仪式感与合规感兼具 |

### 红色使用的克制体现

```
修改前单屏红色面积估算：
├── 导航栏背景：#C62828 → 约 8%
├── 状态标签（全红）：约 3%
├── 主按钮（红色）：约 4%
├── Tab 选中态：约 2%
└── 总计：约 17% ← 视觉侵略性过强

修改后单屏红色面积估算：
├── 品牌竖线：约 0.01%
├── 未读角标：约 0.1%
├── 待出厂状态标签：约 0.2%
├── 危险操作按钮（作废）：约 1.5%
└── 总计：约 1.8% ← 严格控制在 3% 以内
```
