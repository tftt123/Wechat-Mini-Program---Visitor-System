# 访客管理系统 - 超详细部署指南（零基础版）

本文档面向没有任何微信小程序开发经验的用户，手把手教你从零开始部署上线。

---

## 第一步：准备工作（5分钟）

### 1.1 需要准备的材料

| 材料 | 说明 |
|------|------|
| 营业执照（企业/个体户） | 小程序注册必须用企业主体 |
| 一个未注册过微信公众平台的邮箱 | 推荐使用企业邮箱或 QQ 邮箱 |
| 管理员手机号 | 用于接收验证码 |
| 管理员身份证 | 需要实名认证 |
| 一台 Windows/Mac 电脑 | 用于安装开发工具 |

> **注意**：个人主体无法使用云开发的部分高级功能，且不能发布正式版，所以必须用企业/个体户主体注册。

### 1.2 建议的测试手机号

部署完成后测试时，建议准备 3 个不同的手机号：
- 1 个注册为**员工**
- 1 个注册为**门卫**
- 1 个注册为**人事**

---

## 第二步：注册微信小程序账号（10分钟）

### 2.1 访问微信公众平台

1. 打开浏览器，访问：https://mp.weixin.qq.com/
2. 点击页面右上角的 **"立即注册"**
3. 选择账号类型：**小程序**

### 2.2 填写注册信息

1. **填写邮箱**：输入你的邮箱地址
2. **激活邮箱**：登录邮箱查看验证码，填入
3. **信息登记**：
   - 主体类型选择 **"企业"** 或 **"个体工商户"**
   - 填写企业名称、营业执照注册号
   - 管理员信息（姓名、身份证号、手机号）
   - 用管理员微信扫码验证身份
4. **提交审核**：通常 1-3 个工作日内通过

### 2.3 获取 AppID

注册完成后：

1. 登录小程序后台：https://mp.weixin.qq.com/
2. 在左侧菜单找到 **"开发" → "开发管理"**
3. 点击顶部 **"开发设置"** 标签页
4. 找到 **"AppID(小程序ID)"**，复制这个字符串（格式类似：`wx1234567890abcdef`）
5. 先保存在记事本里，后面会用到

---

## 第三步：开通微信云开发（5分钟）

云开发是腾讯云提供的服务器less后端服务，不需要你自己买服务器。

### 3.1 开通云开发

1. 登录小程序后台：https://mp.weixin.qq.com/
2. 在顶部菜单找到 **"云开发"** 按钮，点击进入
3. 点击 **"开通"**
4. 选择环境：
   - **环境名称**：填写 `visitor-system`（随意）
   - 点击 **"创建环境"**
5. 稍等片刻，环境创建完成

### 3.2 获取云开发环境ID

1. 进入云开发控制台后，在左上角可以看到一个下拉框
2. 下拉框里显示的就是你的 **环境ID**，格式类似：`visitor-system-xxx`
3. 复制这个环境ID，和 AppID 一起保存在记事本里

> **这两个信息非常重要，后面都要用到！**

---

## 第四步：安装微信开发者工具（10分钟）

### 4.1 下载安装

1. 访问下载页面：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 根据你的电脑系统选择对应版本：
   - Windows 64位 → 选 `Windows 64位` 版本
   - Mac → 选 `macOS` 版本
3. 下载完成后双击安装，一直点"下一步"即可

### 4.2 首次启动

1. 打开安装好的 **"微信开发者工具"**
2. 用手机微信扫码登录
3. 登录成功后进入主界面

---

## 第五步：导入项目（5分钟）

### 5.1 修改项目配置

在导入项目之前，需要先修改两个配置文件中的关键信息。

#### 修改 AppID

1. 找到项目文件夹：`C:\Users\adm\visitor-system\`（就是刚才生成的项目）
2. 用记事本（或任意文本编辑器）打开 `project.config.json`
3. 找到 `"appid": "wxYOURAPPID"`
4. 把 `wxYOURAPPID` 替换成你刚才注册的 **小程序 AppID**

修改前：
```json
"appid": "wxYOURAPPID",
```

修改后（示例）：
```json
"appid": "wx1234567890abcdef",
```

#### 修改云开发环境ID

1. 用记事本打开 `miniprogram\app.js`
2. 找到 `env: 'your-cloud-env-id'`
3. 把 `'your-cloud-env-id'` 替换成你的 **云开发环境ID**

修改前：
```javascript
env: 'your-cloud-env-id',
```

修改后（示例）：
```javascript
env: 'visitor-system-7g8h9j0k1',
```

### 5.2 导入到开发者工具

1. 打开微信开发者工具
2. 点击 **"导入项目"**（如果是首次打开，点击左侧的 `+` 号）
3. 在弹出的窗口中：
   - **目录**：点击选择文件夹，找到 `C:\Users\adm\visitor-system\`，选中它
   - **AppID**：选择 **"测试号"** 或填入你注册的小程序 AppID
   - **后端服务**：勾选 **"微信云开发"**
   - **项目名称**：填写 `访客管理系统`
4. 点击 **"确定"**

### 5.3 首次加载

导入成功后，你会看到左侧是代码编辑器，中间是模拟器（一个手机界面），右侧是调试器。

第一次加载可能会比较慢，请耐心等待。

---

## 第六步：创建数据库集合（10分钟）

数据库集合相当于 Excel 表格，需要手动创建。

### 6.1 打开云开发控制台

1. 在开发者工具的顶部工具栏，点击 **"云开发"** 按钮
2. 会弹出一个网页，这就是 **云开发控制台**

### 6.2 创建集合

在左侧菜单点击 **"数据库"**，然后点击 **"+ 添加集合"**，依次创建以下 4 个集合：

1. `users` —— 存储员工/人事/门卫账号
2. `visits` —— 存储拜访记录
3. `visit_logs` —— 存储操作日志
4. `notifications` —— 存储通知消息

创建方法：
- 在数据库页面，找到 "+ 添加集合" 按钮
- 输入集合名称，按回车或点击确定
- 重复 4 次，创建完 4 个集合

### 6.3 创建索引（重要！）

索引能大幅提升查询速度，否则查询会很慢。

#### 给 users 集合创建索引

1. 点击 `users` 集合进入
2. 点击顶部的 **"索引管理"** 标签
3. 点击 **"添加索引"**
   - 索引名称：`phone_unique`
   - 索引属性：`唯一`
   - 索引字段：
     - 字段名：`phone`
     - 排序：`升序`
   - 点击 **"确定"**

#### 给 visits 集合创建索引

依次创建以下索引：

| 索引名称 | 索引属性 | 字段名 | 排序 |
|---------|---------|-------|------|
| `inviteCode_unique` | 唯一 | `inviteCode` | 升序 |
| `employeeId_index` | 非唯一 | `employeeId` | 升序 |
| `status_index` | 非唯一 | `status` | 升序 |
| `visitDate_index` | 非唯一 | `visitDate` | 升序 |

创建方法同上，在 visits 集合的索引管理中添加。

#### 给 notifications 集合创建索引

1. 进入 `notifications` 集合
2. 添加索引：
   - 索引名称：`recipientId_index`
   - 索引属性：非唯一
   - 字段名：`recipientId`
   - 排序：升序

> **注意**：`visit_logs` 集合不需要额外索引。

---

## 第七步：部署云函数（10分钟）

云函数是后端代码，需要上传到腾讯云才能运行。

### 7.1 设置云开发环境

1. 在开发者工具左侧文件树中，展开 `cloudfunctions`
2. 展开 `auth`
3. 右键点击 `auth` 文件夹
4. 在弹出的菜单中选择 **"创建并部署：云端安装依赖"**
5. 稍等片刻，等待部署完成（底部控制台会显示进度）

### 7.2 部署其他云函数

按照上面的方法，依次右键部署以下文件夹：

- `cloudfunctions/invites` → 创建并部署：云端安装依赖
- `cloudfunctions/visits` → 创建并部署：云端安装依赖
- `cloudfunctions/notifications` → 创建并部署：云端安装依赖

### 7.3 验证部署

部署完成后，可以在云开发控制台 → 云函数中查看，应该能看到 4 个云函数都已上线。

---

## 第八步：配置权限（5分钟）

数据库默认不允许外部访问，需要修改权限。

### 8.1 修改数据库权限

对每个集合都需要设置权限：

1. 在云开发控制台 → 数据库中，点击 `users`
2. 点击顶部的 **"权限设置"**
3. 选择 **"所有用户可读，仅创建者可读写"**（或者选择 **"所有用户可读，所有用户可写"**，部署调试阶段更方便）
4. 点击 **"确定"**

依次对以下 4 个集合设置相同的权限：
- `users`
- `visits`
- `visit_logs`
- `notifications`

> **安全提示**：正式上线后，建议改为"所有用户可读，仅创建者可读写"，并在云函数中校验权限。

## 第九步：测试运行（15分钟）

### 9.1 注册测试账号

1. 在开发者工具中间的模拟器中，点击底部 **"我的"** 标签
2. 点击 **"注册账号"**
3. 填写信息：
   - 手机号：`13800138001`（测试用，随意填写）
   - 姓名：`张三`
   - 角色：`员工`
   - 部门：`生产部`
4. 点击 **"注册"**
5. 用同样的方法再注册一个门卫账号和一个 HR 账号

### 9.2 测试完整流程

#### 场景一：员工生成邀请码

1. 切换到员工账号
2. 点击底部 **"员工端"** 标签
3. 点击 **"生成邀请码"**
4. 填写来访日期、时间、目的
5. 点击生成，记住显示的 6 位数字邀请码

#### 场景二：访客登记

1. 在开发者工具中，点击顶部工具栏的 **"编译"** 按钮（或者按 Ctrl+S）
2. 这会重新加载小程序
3. 由于访客不需要登录，直接模拟扫码：
   - 在模拟器中，点击 **"首页"**
   - 输入刚才的 6 位邀请码
   - 点击进入系统
   - 填写访客姓名、手机号、公司等
   - 点击提交

#### 场景三：员工确认

1. 切换到员工账号
2. 在 **"待确认"** 列表中，应该能看到刚提交的拜访
3. 点击进入详情
4. 点击 **"确认拜访"**

#### 场景四：门卫确认入厂

1. 切换到门卫账号
2. 点击底部 **"门卫端"** 标签
3. 在 **"待入厂"** 列表中，应该能看到这条记录
4. 点击进入详情
5. 点击 **"确认入厂"**

#### 场景五：物品确认

1. 切换到员工账号
2. 在今日来访中找到该记录（状态为"已入厂"）
3. 点击进入详情
4. 点击 **"不携带物品"** 或 **"携带物品"**

#### 场景六：门卫确认出厂

1. 切换到门卫账号
2. 在 **"待出厂"** 列表中，应该能看到这条记录
3. 点击进入详情
4. 点击 **"确认出厂"**
5. 流程结束！

## 第十步：发布上线（可选，审核需1-3天）

### 10.1 生成固定二维码

1. 登录小程序后台：https://mp.weixin.qq.com/
2. 左侧菜单：**"工具" → "生成小程序码"**
3. 选择 **"小程序码"**（不是普通二维码）
4. **页面路径**：填写 `pages/index/index`
5. 点击 **"生成"**
6. 下载图片，打印后贴在门卫处

### 10.2 提交审核

1. 在开发者工具中，点击顶部工具栏的 **"上传"** 按钮
2. 填写版本号（如 `1.0.0`）和项目备注
3. 点击上传
4. 登录小程序后台 → **"管理" → "版本管理"**
5. 在"开发版本"中找到刚上传的版本
6. 点击 **"提交审核"**
7. 填写审核信息，提交
8. 等待 1-3 个工作日审核通过
9. 审核通过后，点击 **"发布"** 即可上线

## 常见问题

### Q1：提示"云开发未初始化"或"云函数调用失败"

**原因**：app.js 中的环境ID 填错了，或者云函数没有部署成功。

**解决**：
1. 检查 `app.js` 中的 `env` 是否和你的云开发环境ID一致
2. 检查 4 个云函数是否都已部署（云开发控制台 → 云函数）
3. 如果修改了 app.js，记得保存后点击开发者工具的"编译"按钮

### Q2：提示"集合不存在"

**原因**：数据库集合没有创建。

**解决**：按照第六步的方法，创建 4 个集合。

### Q3：注册时提示"该手机号已注册"

**原因**：同一个手机号在同一个集合中只能注册一次。

**解决**：换一个手机号测试，或者去数据库中手动删除已有记录。

### Q4：注册时提示"该手机号未注册"

**原因**：登录时使用的手机号没有在系统中注册。

**解决**：先注册再登录。

### Q5：门卫端看不到任何记录

**原因**：
1. 记录状态不是 `confirmed`、`checked_in` 或 `item_checked`
2. 记录的 `visitDate` 不是今天

**解决**：
1. 检查员工是否已经点击了"确认拜访"
2. 检查拜访日期是否是今天

### Q6：员工端"待确认"列表为空

**原因**：访客还没有提交信息，或者员工不是该条记录的被访人。

**解决**：
1. 确认访客已经通过邀请码提交了信息
2. 确认当前登录的员工是创建该邀请码的人

## 进阶配置

### 添加短信验证码（生产环境建议）

当前版本为了简化部署，使用手机号直接登录。生产环境建议接入短信验证码：

1. 登录小程序后台 → 左侧 **"开发" → "开发管理" → "开发设置"**
2. 找到 **"服务器域名"**，添加你的短信服务商 API 域名
3. 修改 `cloudfunctions/auth/src/index.js` 中的 `login` 函数，增加验证码校验逻辑
4. 重新部署 `auth` 云函数

### 添加订阅消息通知

1. 小程序后台 → **"功能" → "订阅消息"**
2. 点击 **"公共模板库"**，搜索"访客"
3. 选择合适的模板，点击 **"选用"**
4. 记录模板ID
5. 修改 `cloudfunctions/visits/src/index.js` 中的通知逻辑，加入 `wx.cloud.callFunction` 调用订阅消息发送
